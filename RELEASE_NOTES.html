ACE View is an ontology and rule editor that uses Attempto Controlled English (ACE) in order to create, view and edit OWL ontologies and SWRL rulesets. For more information, see: http://attempto.ifi.uzh.ch/aceview/.

Version 1.3.0 (for Protege 4.1 Build 213+). Release date: 2010-11-09

This version works with Protege 4.1, and is not compatible with any earlier version of Protege.

Note that ACE View 1.3.x should be considered a development branch, some features present in 1.2.x are not available here yet.

Update URL: http://aceview.googlecode.com/svn/branches/1_3/update.properties.

  • ADDED: New preference "Use the lexicon of morphological annotations" (ON by default), which can be switched OFF in order not to generate morph annotations to OWL entities. In this case each entity must be referenced from an ACE sentence via its lemma form. This might be useful when loading exisitng OWL ontologies where entity names are not standard English words.
  • ADDED: Q&A View has now a checkbox "Search answers" if set then highlights/shows only the questions whose _answer_ contains the selected entity. (By default shows the questions that contain the selected entity.)
  • ADDED: Action "Annotate every axiom with ACE text".
  • ADDED: Some additional examples to the documentation page.
  • IMPROVED: Loading ontologies with lots of data property axioms is now faster.
  • IMPROVED: Q&A View now shows structured entities as answers, e.g. equivalent classes are grouped together.
  • IMPROVED: ACE Word Usage View now displays more information about the entity itself (IRI, rendering, annotations).
  • UPDATED: all external libraries.
  • CHANGED: Not annotating the axioms anymore with ACE text when the axioms are added, but there is a new Action which can be used e.g. before saving the ontology.
  • REMOVED: AceWiki export. Will be added back in a future version.

Version 1.2.20 (Protege 4.0.113 (r14610, OWL API 2.2.1.1138)). Release date: 2009-06-22

  • IMPROVED: In ACETextViewComponent, replaced SwingWorker with the new Protege BackgroundTask. This gives a nicer integration with Protege.
  • IMPROVED: Removed the path of the SWI-Prolog executable from the preferences and the preferences panel. This is not needed as libjpl knows this path.
  • IMPROVED: pretty-printing of Saxon genitives (e.g. Polyneikes'child -> Polyneikes' child)
  • CHANGED: The toString-serialization of an ACE text is now a \n\n-connected toString-serializations of its snippets. The ACE text view reflects this, i.e. no longer are sentences added and removed in this view. Instead, snippets are added and removed. This aligns nicely with the new paragraph support in ACE/APE 6.5.
  • CHANGED: Renamed: ace:Superman -> ace:Universe; ace:know -> ace:contain. (These entities are generated in the background to capture the semantics of sentences like "There are exactly 27 EU-countries.")
  • FIXED: Some preferences were not remembered
  • UPDATED: google-collect to v1.0 RC2
  • UPDATED: swingx to v1.0

Version 1.2.13 (Protege b104 (r12284, OWL API r962)). Release date: 2008-11-26

  • Now updatable via the P4 plug-in update system
  • ADDED: "Why?" button to Snippet Editor. Triggers an explanation of the selected snippet, if the reasoner is switched on. For an asserted snippet, it provides a way to find out if the snippet is logically redundant. This is marked by the existence of more than one explanation.
  • ADDED: Similar snippets list to Feedback view. Shows snippets that are similar (contain the same content words) to the selected snippet.
  • ADDED: Explanation view (previously was included in Entailments View)
  • ADDED: Some ACE View related links to the Help-menu
  • IMPROVED: Q&A view: Clicking on an answer generates a full answer into the Snippet Editor. E.g. for the question "Which EU-country borders a NATO-country?" and the answer "Estonia", the full answer is "Estonia is a EU-country that borders a NATO-country.". This generated snippet can be explained by clicking on the Snippet Editor's "Why?" button.
  • IMPROVED: ACE View preferences panel
  • CHANGED: The default OWL-to-ACE service is now accessible via the standard port 80.
  • FIXED: All views were reacting to the selected snippet event. For many views this was unnecessary and slowed down browsing large ontologies.
